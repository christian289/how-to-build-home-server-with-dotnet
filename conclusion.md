# 맺음말

축하합니다! N100 미니 PC로 시작하는 홈서버 구축 여정을 함께 해주셔서 감사합니다.

## 지속적인 학습의 중요성

홈서버는 단순히 서비스를 운영하는 것 이상의 의미를 가집니다. 이것은 끊임없는 학습과 실험의 플랫폼입니다.

### 배우게 되는 것들

홈서버를 운영하면서 여러분은 다음과 같은 귀중한 경험을 쌓게 됩니다:

**인프라 관리**
- Linux 시스템 관리
- 네트워크 구성 및 문제 해결
- 보안 설정과 취약점 대응

**컨테이너 기술**
- Docker와 Docker Compose 활용
- 서비스 격리와 의존성 관리
- 쿠버네티스 오케스트레이션 (K3s)

**개발 스킬**
- .NET, Python, Node.js 등 다양한 언어 실습
- 데이터베이스 설계와 최적화
- API 개발과 배포
- CI/CD 파이프라인 구축

**문제 해결 능력**
- 로그 분석과 디버깅
- 성능 병목 찾기와 최적화
- 시스템 장애 대응

### 실패는 최고의 스승

홈서버를 운영하다 보면 많은 시행착오를 겪게 됩니다:
- 컨테이너가 갑자기 멈추는 경우
- 디스크 공간이 부족한 상황
- 네트워크 설정 오류
- 보안 침해 시도

하지만 이러한 문제들을 직접 해결하면서 실력이 향상됩니다. **실패를 두려워하지 마세요.** 홈서버는 안전한 실험실입니다. 최악의 경우 처음부터 다시 시작하면 됩니다.

이 책을 마쳤다면 다음 단계를 고려해보세요:

### 3. 실제 프로덕션 서비스 운영

**사이드 프로젝트**

## 마지막 조언

홈서버 여정은 마라톤이지 단거리 달리기가 아닙니다. 다음 원칙을 명심하세요:

### 1. 작게 시작하고 점진적으로 확장하라
처음부터 모든 것을 완벽하게 구축하려고 하지 마세요. 기본적인 서비스부터 시작하여 필요에 따라 확장하세요.

### 2. 문서화하라
설정한 내용을 기록하세요. 몇 달 후에는 왜 그렇게 설정했는지 기억하지 못할 수 있습니다.

### 3. 백업은 필수
"백업이 없다면 데이터가 중요하지 않다는 의미다"라는 말을 항상 기억하세요. 3-2-1 백업 규칙을 따르세요.

### 4. 보안을 우선시하라
홈서버라고 해서 보안을 소홀히 해서는 안 됩니다. 정기적으로 업데이트하고, 강력한 비밀번호를 사용하며, 불필요한 포트는 닫아두세요.

### 5. 즐거움을 잃지 마라
홈서버는 취미입니다. 스트레스를 받지 말고 재미있게 즐기세요. 새로운 기술을 배우고, 실험하고, 때로는 완전히 망가뜨려도 괜찮습니다.
